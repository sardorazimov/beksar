# Beksar.AI — React Frontend

This directory contains the React 19 + TypeScript frontend for Beksar.AI.

---

## Pages

| Route | File | Description |
|-------|------|-------------|
| `/` | `Landing.tsx` | Welcome page with feature highlights and CTA buttons |
| `/login` | `Login.tsx` | Firebase email/password authentication (sign up or log in) |
| `/agent` | `Agent.tsx` | Main AI agent interface |

---

## Agent Page (`Agent.tsx`)

The agent page is the core of the app. It provides:

### Layout

```
┌─────────────────────────────────────────────────────┐
│  Drag region (frameless window)                      │
├──────────────┬──────────────────────────────────────┤
│              │  Toolbar (provider, live indicators)  │
│   Sidebar    ├──────────────────────────────────────┤
│  (nav/user)  │  Activity panel (collapsible)         │
│              ├──────────────────────────────────────┤
│              │  Chat messages                        │
│              ├──────────────────────────────────────┤
│              │  Input (textarea + send button)       │
└──────────────┴──────────────────────────────────────┘
```

### State

| State | Type | Purpose |
|-------|------|---------|
| `messages` | `Message[]` | Committed chat history |
| `response` | `string` | Accumulates streaming tokens |
| `isStreaming` | `boolean` | Disables input while streaming |
| `isScreenLive` | `boolean` | Activates live screen mode |
| `isScreenPaused` | `boolean` | Pauses the live session |
| `isChessMode` | `boolean` | Chess analysis focus |
| `screenActivities` | `ScreenActivity[]` | Live activity feed (last 20 entries) |
| `chessAnalysis` | `string` | Current move suggestion shown in toolbar |
| `activeProvider` | `Provider` | `"openai" \| "anthropic" \| "gemini"` |
| `apiKeys` | `Record<Provider, string>` | Keys stored in localStorage |

### Tauri Events

| Event | Direction | Description |
|-------|-----------|-------------|
| `stream_chunk` | Backend → Frontend | Appended to `response` state |
| `stream_end` | Backend → Frontend | Commits `response` to `messages` |

### Tauri Commands

| Command | Arguments | Description |
|---------|-----------|-------------|
| `stream_prompt` | `prompt`, `apiKey`, `provider` | Triggers AI streaming response |

---

## Key Design Decisions

- **Streaming display** — The `response` state is rendered live inside the streaming bubble alongside a blinking cursor. When `stream_end` fires, it moves to the committed `messages` array.
- **Live screen simulation** — Screen activity entries are generated by `setInterval` when `isScreenLive` is true and `isScreenPaused` is false. In a future release, real screen captures will be sent to the AI via Tauri's screenshot API.
- **BYOK** — API keys are never sent to any backend. They go directly from the browser to the AI provider via the Tauri Rust layer.
- **Frameless window** — A `data-tauri-drag-region` div at the top allows dragging the window without native title bar buttons.

## Development

```bash
# Start Vite dev server only (no Tauri window)
npm run dev

# Start Tauri dev (frontend + native window)
npm run tauri dev

# Type check
npx tsc --noEmit

# Production build
npm run build
```
